How it works

a two-component injection chain

01
C# Loader
A lightweight WPF launcher finds the javaw.exe process and auto-selects bridge.dll (1.8.9) or bridge_121.dll (1.21.11) via CreateRemoteThread + LoadLibraryA.
‚Üí
02
C++ Bridge DLL
Attaches to the JVM via JVMTI & JNI. Dynamically discovers Minecraft classes at runtime ‚Äî no hardcoded obfuscated names, works across Lunar versions. Hooks SwapBuffers to render overlays every frame.
‚Üí
03
TCP Bridge
Game state is streamed to the loader over TCP port 25590. Config + module keybind state are pushed back continuously. Zero packet modification ‚Äî purely read-only game state.

Requirements & Setup

Requirements

  • ‚óè Windows 10/11 x64

  • ‚óè Lunar Client (Java 1.8.9 or 1.21.11)

Lunar Client JVM Flags

Add these to Custom JVM Arguments in Lunar Client settings:

-XX:+EnableDynamicAgentLoading -XX:-DisableAttachMechanism

Usage

  1. Launch Lunar Client with JVM flags
  2. Run LegoClicker.exe
  3. Click Inject / Connect
  4. Use Insert for 1.8.9 internal ClickGUI
  5. For 1.21.11, use the external control center window
  6. Press backtick (` ) to toggle autoclicker
